项目创建向导（五步流程）

1
信息录入
项目名称、代号、简介、起止时间、预算、风险等级
2
类别选择
单机/模块/软件/技术开发等类别，决定后续流程模板
3
流程绑定
确认/调整流程模板，预览完整活动图
4
团队分配
指定负责人，系统推荐候选人并提示负荷
5
计划确认
查看甘特图，调整里程碑，正式创建项目

工作台功能（类似飞书工作台）

  • • 可拖拽卡片式布局
  • • 支持自定义排序
  • • 支持显隐控制
  • • 常用功能快速访问

个人待办与通知

  • • 时间轴式任务聚合
  • • 多渠道消息推送
  • • 待办事项提醒
  • • 审批流程通知

统一入口导航

  • • 沉浸式深色顶部导航
  • • 可折叠左侧边栏
  • • 自适应主内容区
  • • 面包屑导航

技术架构要求

架构原则

微前端架构

核心模块可独立放在架构中，支持多团队独立开发，提高开发效率

离线局域网部署

纯内网部署，物理隔离，零依赖外部CDN，数据不出机房

不使用虚拟化技术

项目不使用 Docker 和 K8S 这类虚拟技术

技术栈选型

Vue.js 3
前端框架（参考 vue-element-admin）
Element Plus
UI组件库
qiankun
微前端框架
Spring Boot
后端服务
PostgreSQL
数据存储
